29 /*
30  * Kafka-connector specific configs
31  * -----------------------------------------------------------------------
32  * Description: These configuration options are additional to the librdkafka ones. Options
33  * pertaining to a producer start with 'internal.producer' and for topics start with 'internal.topic'.
34  * Default values are marked with a * or are in {}.
35  *
36  * internal.producer.timeout.ms: {1000} Sets the timeout on poll and flush operations.
37  *
38  * internal.producer.retries: {0} Sets the number of times to retry sending a message from the internal queue before
39  * giving up. Note that this setting is independent of the rdkafka setting 'message.send.max.retries'. This setting
40  * applies to each individual message and not to entire batches of messages which 'message.send.max.retries' does.
41  *
42  * internal.producer.payload.policy: {'passthrough',*'copy'}. Sets the payload policy on the producer.
43  * For sync producers, use 'passthrough' for maximum performance.
44  *
45  * internal.producer.preserve.message.order: {'true',*'false'}. Set to 'true' if strict message order must be preserved.
46  * When setting this, rdkafka option 'max.in.flight' will be set to 1 to avoid possible reordering of packets.
47  * Messages will be buffered and sent sequentially, waiting for broker acks before the next one is processed.
48  * This ensures delivery guarantee at the cost of performance. The internal buffer queue size can be
49  * checked via ProducerMetadata::getInternalQueueLength().
50  * An intermediate, solution with better performance would be to set this setting to 'false' and set 'max.in.flight=1'
51  * which will skip internal buffering altogether. This will guarantee ordering but not delivery.
52  *
53  * internal.producer.max.queue.length: {10000} Maximum size of the internal queue when producing asynchronously
54  * with strict order preservation.
55  *
56  * internal.producer.wait.for.acks: {'true',*'false'} If set to true, then the producer will wait up to the
57  * timeout specified when producing a message via send() or post().
58  *
59  * internal.producer.wait.for.acks.timeout.ms: {0} Set the maximum timeout for the producer to wait for broker acks
60  * when producing a message via send() or post(). Set to -1 for infinite timeout.
61  *
62  * internal.producer.flush.wait.for.acks: {'true',*'false'} If set to true, then the producer will wait up to the
63  * timeout specified when flushing the internal queue.
64  *
65  * internal.producer.flush.wait.for.acks.timeout.ms: {0} Set the maximum timeout for the producer to wait for broker acks
66  * when flushing the internal queue (see above). Set to -1 for infinite timeout.
67  *
68  * internal.producer.log.level: {'emergency','alert','critical','error','warning','notice',*'info','debug'}.
69  * Sets the log level for this producer. Note that if the rdkafka 'debug' property is set, internal.producer.log.level
70  * will be automatically adjusted to 'debug'.
71  *
72  * internal.producer.skip.unknown.headers: {*'true','false'}. If unknown headers are encountered (i.e. for which there
73  * is no registered serializer), they will be skipped. If set to false, an error will be thrown.
74  *
75  * internal.producer.auto.throttle: {'true',*'false'}. When enabled, the producers will be automatically paused/resumed
76  * when throttled by the 'throttle time x multiplier'.
77  *
78  * internal.producer.auto.throttle.multiplier: {1}. Change this value to pause the producer by
79  * 'throttle time x multiplier' instead. This only works if 'internal.producer.auto.throttle=true'.
80  *
81  * internal.producer.queue.full.notification: {'edgeTriggered',*'oncePerMessage','eachOccurence'}.
82  * When registering a QueueFullCallback, this setting determines when the callback will be raised. If set to
83  * 'edgeTriggered', the callback will be raised once after which the application must reset the edge via
84  * ProducerManager::resetQueueFullTrigger() in order to re-enable it again.
85  *
86  */

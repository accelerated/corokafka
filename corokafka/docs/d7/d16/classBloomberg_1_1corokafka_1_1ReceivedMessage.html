◆ commit()

template<typename K , typename P >
void Bloomberg::corokafka::ReceivedMessage< K, P >::commit ( void *  opaque = nullptr)

Commits message with the retry strategy specified in the config for this topic.

Parameters
opaqueApplication-specific pointer which will be returned inside the offset commit callback.
Remarks
This call is blocking until it succeeds or the last retry failed.
If commit() is not called explicitly, it will be called by ~ReceivedMessage() if 'internal.consumer.auto.offset.persist=true' and 'internal.consumer.auto.offset.persist.strategy=commit'.
The application must set 'enable.auto.offset.store=false' and 'enable.auto.commit=false'.

◆ store_offset()

template<typename K , typename P >
void Bloomberg::corokafka::ReceivedMessage< K, P >::store_offset ( void *  opaque = nullptr)

Stores the message offset locally in rdkafka. The offset will be committed when the commit scheduler runs every 'auto.commit.interval.ms'.

Parameters
opaqueApplication-specific pointer which will be returned inside the offset commit callback.
Remarks
If store_offset() is not called explicitly, it will be called by ~ReceivedMessage() if 'internal.consumer.auto.offset.persist=true' and 'internal.consumer.auto.offset.persist.strategy=store'.
The application must set 'enable.auto.offset.store=false', 'enable.auto.commit=true' as well as set a proper value for 'auto.commit.interval.ms'.

◆ enableMessageFanout()

void Bloomberg::corokafka::ProducerManager::enableMessageFanout ( bool  value)

Enables or disables parallel message fan-out to IO threads. Disabled by default.

Note
When enabled messages are distributed among all IO threads on a per-topic basis. When disabled, messages are produced sequentially after serialization.
This setting only affects async producers (i.e. post() methods). This method can be set/reset anytime.
Warning
This should be typically used when some producers get backed-up, in which case posting the messages sequentially would also impact other producers. The fan-out lets other producers work independently.

Asynchronous send. No message delivery guarantee is made and messages are sent in batches unless strict ordering is needed.

Template Parameters

PThe Payload type for this message.
Parameters
topicThe topic to publish to.
keyThe message key.
payloadThe message payload.
headersThe header pack for this message.
opaqueAn opaque data pointer which will be returned inside the delivery callback or the returned future.
Remarks
To guarantee strict message ordering, set 'internal.producer.preserve.message.order = true' which will also set the rdkafka option 'max.in.flight = 1' as it may cause re-ordering or packets.
A message delivery can be tracked by registering a delivery report callback or by blocking on the returned future. Note that both these methods can be used jointly if needed.
Returns
A future containing a message delivery report.
Warning
This method will make an extra copy of the message.

Synchronous send. When this function returns, the message has been ack-ed by N broker replicas based on the broker settings for this topic and the final ack response has been received by this library.

Template Parameters
KThe Key type used in selecting the partition where this message will be sent.
PThe Payload type for this message.
Parameters
topicThe topic to publish to.
keyThe message key.
payloadThe message payload.
headersThe header pack for this message.
opaqueAn opaque data pointer which will be returned inside the delivery callback.
Returns
The number of bytes sent.
Remarks
If the application uses only synchronous sends, better performance can be achieved by setting 'internal.producer.payload.policy = passthrough', which will prevent the payload from being copied inside librdkafka.
To guarantee strict message ordering, set 'internal.producer.preserve.message.order = true' which will also set the rdkafka option 'max.in.flight = 1' as it may cause re-ordering or packets.
